HTML 5 Semantic Tags

NOTE: Not all the HTML 5 and CSS 3 features are supported on all browsers

Semantic elements don't inpact the document flow and have no styleistic difference

Article: The <article> element represents a sectio of content that forms an independent part of a document, such as a blog
post, article, or other self-contained unit of information, that may be linked to or included in some other content body.

Aside: represents a section of the web page that encloses content which is tangentially related to the content around it.

Header: represents the header of a document or a section. A header should contain title and heading information about the related content.

Footer: represents the footer of a document or a section. A footer typically contains information about the author of the document, copyright information, links to related documents etc.
